## Quran Visual Reader — Animated Surah Experience

A mobile-first PWA where readers experience each Surah as an AI-generated animated scene with Arabic recitation, translated narration, and synchronized subtitles. Includes search, bookmarks, and an admin generator to grow the animated library over time.

### Experience overview

1. **Home** — Surah list (1-114) with Arabic name, English name, verse count, and a small thumbnail. Badge shows "Animated" for ready Surahs.
2. **Surah player** — Full-screen vertical scene. Verse-by-verse animated background (AI-generated images with Ken Burns / cross-fade), Arabic text and translation as subtitles, Arabic recitation playing, then translated narration. Tap-to-pause, swipe for prev/next verse, progress bar.
3. **Search** — Search Surah names (Arabic / English / transliteration) and verse text in any installed translation. Tapping a result jumps directly to that verse in the player.
4. **Bookmarks** — Save the current verse with one tap. Bookmark list with Surah name, verse number, snippet, and "resume" button.
5. **Settings** — Translation language picker (English, Urdu, Indonesian, Turkish), reciter picker, autoplay toggle.
6. **Admin / Generator** — Protected page where you select a Surah and click Generate. The system creates a scene per verse (image + narration), stores them, and marks the Surah animated.

### Religious & content guardrails

- **Prophet Muhammad (PBUH)** — never depicted, never narrated by a character voice. His name appears in subtitles with "ﷺ". Verses mentioning him use symbolic imagery (light, the city of Madinah, calligraphy) and the translated narration uses a neutral narrator voice.
- **Other prophets and figures** — depicted respectfully per user choice (full characters allowed).
- **AI image prompts** are templated with strict negative prompts: no faces of Prophet Muhammad, no Kaaba interior, no Allah depiction, modest dress, reverent tone. Each prompt is reviewed by an LLM safety pass before image generation.
- **Source text** — Uthmani Quran text from a trusted public dataset (AlQuran.cloud / Tanzil), not generated by AI.

### Data sources

- **Quran text + translations + audio**: AlQuran.cloud API (free, no key) — provides Arabic Uthmani, multiple translations, and verse-level recitation MP3s from major reciters (Mishary Alafasy, Sudais, Husary, etc.).
- **Translated narration**: ElevenLabs TTS (multilingual v2) generated on-demand and cached.
- **Scene images**: Lovable AI Gateway (Gemini 2.5 Flash Image / Nano Banana) — one image per verse, cached.

### Pages / routes

- `/` — Surah list + search bar
- `/search` — full search results
- `/surah/$number` — animated player (deep-linkable to `/surah/2?verse=255`)
- `/bookmarks` — saved verses
- `/settings` — language, reciter, autoplay
- `/admin` — generator dashboard (auth-gated)
- `/auth` — sign in / sign up (for bookmarks + admin)

### Backend (Lovable Cloud)

Tables:
- `surahs` (114 rows seeded) — number, name_ar, name_en, name_translit, verse_count, revelation_place, is_animated
- `verses` — surah_number, verse_number, text_ar, juz, page, mentions_prophet_muhammad (bool, precomputed)
- `translations` — verse_id, language, text
- `scenes` — verse_id, image_url, image_prompt, status (pending/ready/failed)
- `narrations` — verse_id, language, audio_url, voice_id
- `bookmarks` — user_id, surah_number, verse_number, note, created_at
- `user_settings` — user_id, translation_language, reciter, autoplay
- `user_roles` — separate roles table with admin role for generator access

Storage buckets: `scene-images` (public), `narrations` (public).

Server functions: `fetchSurah`, `searchQuran`, `generateSceneForVerse` (admin), `generateSurah` (admin, queues all verses), `getOrCreateNarration`.

### Tech & integrations

- **Lovable Cloud** for DB, auth, storage, RLS
- **Lovable AI Gateway** for image generation + LLM prompt building
- **ElevenLabs** connector for translation TTS (multilingual v2)
- **Public AlQuran.cloud API** for Arabic text + recitation URLs (no key)
- **PWA manifest** (no service worker) so users can install to home screen
- **TanStack Start** server functions for all backend logic

### Visual design

- Mobile-first, dark default theme with warm gold/amber accent
- Arabic typography: "Amiri" or "Scheherazade New" for verse text; Inter for UI
